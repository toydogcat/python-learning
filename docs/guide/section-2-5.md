# 第 2.5 節 – 註解 (Comments)

註解是寫給「人」看的，Python 直譯器在執行程式時會完全忽略註解。

## 2.5.1 為什麼需要註解？
*   **解釋程式碼**：說明某段複雜邏輯的功能。
*   **標註作者與日期**：記錄開發資訊。
*   **暫時停用程式碼**：在測試時，可以將不想要的程式碼標註起來，讓它不執行。

## 2.5.2 如何撰寫註解？
在 Python 中，註解以井字號 `#` 開頭。從 `#` 開始到該行結束的所有內容都會被視為註解。

```python
# 這是單行註解
print("Hello!")  # 這是行尾註解

# 如果要撰寫多行註解
# 您必須在每一行的開頭
# 都加上井字號
```

### 自我註解 (Self-commenting)
好的開發者會透過命名來減少註解的需求。
例如：`square_area` 會比 `a` 更好理解，這種命名方式稱為「自我註解」。

## 2.5.3 快速鍵提示
大多數的編輯器（如 VS Code 或 IDLE）都支援快速鍵來切換註解：
*   **Windows**: `Ctrl` + `/`
*   **Mac**: `Cmd` + `/`
