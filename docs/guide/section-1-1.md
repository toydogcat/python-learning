# 第 1.1 節 – 程式設計簡介

歡迎來到第一模組！我們將從學習程式設計的一些通用概念開始，例如指令列表、來源檔案、語言要素、編譯與直譯。準備好了嗎？讓我們開始吧！

## 電腦程式是如何運作的？

程式讓電腦變得有用。沒有程式，即使是最強大的電腦也只是一個物體。同樣地，沒有演奏者，鋼琴也只是一個木盒子。

電腦能夠執行非常複雜的任務，但這種能力並非與生俱來的。電腦的本質相當不同。

它只能執行極其簡單的操作。例如，電腦本身無法理解複雜數學函數的值。

現代電腦只能評估加法或除法等基本操作的結果，但它們可以做得非常快，並且可以幾乎無限次地重複這些動作。

想像一下，你想知道在一段長途旅行中達到的平均速度。你知道距離，你知道時間，你需要速度。

電腦當然能夠計算這個，但電腦並不知道距離、速度或時間等概念。因此，有必要指示電腦：
1. 接受一個代表距離的數字；
2. 接受一個代表旅行時間的數字；
3. 將前一個值除以後一個值，並將結果儲存在記憶體中；
4. 以可讀的格式顯示結果（代表平均速度）。

這四個簡單的動作就構成了一個**程式**。

## 自然語言 vs. 程式設計語言

語言是表達和記錄思想的手段（和工具）。我們身邊有很多語言。有些既不需要說話也不需要寫作，例如肢體語言。

另一種你每天使用的語言是你的母語。電腦也有自己的語言，稱為**機器語言 (Machine Language)**，它是非常基礎的。

電腦本身沒有任何智慧。你可以說它像一隻訓練有素的狗——它只對一組預先確定的已知命令做出反應。

它能識別的命令非常簡單。我們可以想像電腦對「取得那個數字，除以另一個並保存結果」之類的命令做出反應。

一組完整的已知命令稱為**指令列表 (Instruction List, IL)**。

### 語言的組成要素

我們可以說每種語言（機器語言或自然語言）都包含以下要素：
*   **字母表 (Alphabet)**：用於構建特定語言單詞的一組符號。
*   **詞彙 (Lexis)**：語言提供給其使用者的一組單詞（也稱為字典）。
*   **語法 (Syntax)**：一組用於確定特定單詞序列是否形成有效句子的規則。
*   **語義 (Semantics)**：一組確定特定短語是否有意義的規則。

## 機器語言 vs. 高階語言

指令列表 (IL) 實際上是機器語言的字母表。這是我們可以用來向電腦發出命令的最簡單、最原始的符號集。它是電腦的母語。

我們需要一種人類可以編寫程式、電腦可以執行程式的語言，這種語言比機器語言複雜得多，但又比自然語言簡單得多。

這類語言通常被稱為**高階程式設計語言 (High-level programming languages)**。它們在某種程度上與自然語言相似，因為它們使用人類可讀的符號、單詞和慣例。

使用高階程式設計語言編寫的程式稱為**原始碼 (Source code)**，包含原始碼的檔案稱為**來源檔案 (Source file)**。

## 編譯 vs. 直譯 (Compilation vs. Interpretation)

將程式從高階程式設計語言轉換為機器語言有兩種不同的方式：

1.  **編譯 (Compilation)**：原始程式被翻譯一次（但每次修改原始碼時都必須重複此動作），得到一個包含機器碼的檔案（例如 Windows 下的 `.exe` 檔案）。執行翻譯的程式稱為**編譯器 (Compiler)**。
2.  **直譯 (Interpretation)**：每次執行程式時都可以翻譯原始程式。執行這類轉換的程式稱為**直譯器 (Interpreter)**。這意味著您不能只分發原始碼，因為終端使用者也需要直譯器來執行它。

### 編譯 vs. 直譯的優缺點

| 特性 | 編譯 (Compilation) | 直譯 (Interpretation) |
| :--- | :--- | :--- |
| **優點** | 執行速度通常較快；終端使用者不需要編譯器。 | 完成後即可立即執行；程式碼可在不同架構的電腦上運行。 |
| **缺點** | 編譯過程可能很耗時；需要針對不同硬體平台準備編譯器。 | 執行速度較慢（直譯器會消耗部分效能）；使用者必須擁有直譯器。 |

### 這對您意味著什麼？
*   **Python 是一種直譯式語言**。這意味著它繼承了上述所有的優缺點。
*   如果您想用 Python 編寫程式，您需要 **Python 直譯器**。沒有它，您將無法運行您的程式碼。幸運的是，Python 是免費的。
